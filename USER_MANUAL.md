# Voice Trainer — 使用者說明

**版本：** 1.0　|　**最後更新：** 2026-05-24

---

## 這是什麼？

Voice Trainer 是一個專屬於你的對話風格學習工具。

它的目的是**收集你真實說話的方式**，然後把這個「語音指紋」套用到 Relationship OS 的 AI agent 上，讓 AI 建議的訊息不再像制式的 AI 語氣，而是像你自己在傳訊息。

**你只需要做一件事：** 看到情境提示，用最自然的方式打訊息。AI 在背後幫你分析風格。

---

## 第一次使用設定

### 步驟一：申請 4 個免費 AI 帳號

這些帳號**獨立於其他專案**，不共用。每個大約 2-3 分鐘申請完成。

| 服務 | 申請連結 | 說明 |
|------|---------|------|
| **Groq** | https://console.groq.com | 免費，每天 14,400 次請求，最快 |
| **Cerebras** | https://cloud.cerebras.ai | 免費，wafer-scale 超快推理 |
| **NVIDIA NIM** | https://build.nvidia.com | 免費，登入後點「Get API Key」取得 nvapi-xxx |
| **OpenRouter** | https://openrouter.ai | 免費 tier，使用 `:free` 後綴模型 |

> **Ollama（選用）：** 本地推理，完全免費、私密、不需帳號。  
> 安裝：https://ollama.ai → 安裝後執行 `ollama pull qwen2.5:7b`

---

### 步驟二：填入 API Keys

打開 `~/CloudSync/voice-trainer/.env`，填入你的 keys：

```
VOICE_GROQ_API_KEY=gsk_你的真實key
VOICE_CEREBRAS_API_KEY=csk_你的真實key
VOICE_NVIDIA_API_KEY=nvapi-你的真實key
VOICE_OPENROUTER_API_KEY=sk-or-v1-你的真實key
```

---

### 步驟三：啟動工具

```bash
cd ~/CloudSync/voice-trainer
npm start
```

打開瀏覽器：**http://localhost:3005**

## 日常使用流程

### 收集訓練（最重要的功能）

1. **選左邊類別** — 例如「打招呼」
2. **看情境提示** — 描述了一個真實的對話場景
3. **自然回覆** — 就像真的在傳訊息，不要想太多
4. **按「送出分析」** 或 **⌘↵**
5. 等 5-20 秒 — 最多 5 個 AI 同時分析
6. 看到分析結果後，**自動換下一題**

> **最重要的原則：不要刻意，用最自然的方式打。**  
> 錯別字沒關係、emoji 沒關係、中英混用沒關係，這才是真實的你。

---

### 看風格報告

點上方「📊 風格報告」頁籤，可以看到：

- **正式程度** — 0% = 超口語，100% = 超正式
- **溫暖熱情** — 你表達關心的強度
- **直接程度** — 直接說重點 vs 拐彎抹角
- **幽默感** — 是否帶有玩笑感
- **中文比例** — 中英文混用的比例
- **常用 Emoji** — 你最常用哪些
- **特徵詞彙** — 你的招牌用語
- **開頭/結尾習慣** — 你慣用的句首句尾

---

### 套用到 Relationship OS

收集 **至少 5 個樣本** 後，右上角「⬆ 套用到 Relationship OS」變為可點擊。

點擊後：
1. AI 根據你的所有樣本生成各類別訊息模板
2. 模板顯示在「💬 ROS 模板」頁籤
3. `voice-profile.json` 透過 **Syncthing 自動同步**到 chusMBp
4. Relationship OS 的 AI agent **下次建議訊息時自動套用**你的風格

**無需手動操作，完全自動。**

## 左邊欄 AI 分析器狀態

| 燈號 | 意思 |
|------|------|
| 🟢 綠色 | 有 key，可用 |
| 🔴 紅色 | 未設定 key |

Ollama 沒安裝時顯示紅色但不影響功能，其他 4 個 cloud AI 會補上。

## 與 Relationship OS 的整合

Voice Trainer 透過兩個機制影響 Relationship OS：

### 機制一：自動風格注入

每次你在 Relationship OS 問「我應該怎麼跟某人說」，groqAgent 在建議訊息時會自動讀取你的風格：

```
（模仿語氣：自然隨性，非常熱情溫暖，74% 中文 26% 英文，
 平均 38 字，常用 😊 哈哈 👍）
```

這個 hint 注入到 AI 的提示詞中，讓建議的訊息更像你的口吻。

### 機制二：訊息模板

點「套用到 Relationship OS」後生成的模板，可以直接在 Relationship OS 中使用，適合快速傳固定類型的訊息。

## 常見問題

**Q：分析要等多久？**  
A：最快的 Groq/Cerebras 約 2-3 秒，Gemini 5-8 秒，OpenRouter 最多 20 秒。送出後等到出現分析結果（通常 5-10 秒）。

**Q：API 沒有填，可以用嗎？**  
A：可以看到 UI 和話題，但按「送出分析」會失敗。話題使用預建的 fallback（不需要 AI），只有分析功能需要 key。

**Q：要同時有所有 API key 嗎？**  
A：不需要。有幾個就用幾個，缺少的 AI 自動跳過。但建議至少有 Groq + Cerebras（最穩定的兩個）。

**Q：我的對話記錄有沒有傳到外面？**  
A：傳送到你申請的 API（Groq、Cerebras 等）做分析。如果擔心隱私，可以只用 Ollama（安裝後本地分析，資料不離機）。

**Q：Relationship OS 會馬上改變嗎？**  
A：會。Syncthing 通常在幾秒到幾分鐘內同步，Relationship OS 的 voiceProfile.ts 每 5 分鐘重新讀取一次，所以最多 5 分鐘內生效。

**Q：可以刪掉樣本重來嗎？**  
A：可以。刪除 `data/conversations.json` 和 `data/voice-profile.json` 後重啟，資料清空重新收集。
